Mission Statement

The objective of this project is to take an inherently unsafe environment and secure it for commercial use.

Why Is the OS Set Up This Way?

Red Star OS is intentionally designed to control information:

  • It blocks foreign media, news, and software.
  • Its macOS-like interface is used to project modernity and prestige.
  • The OS includes built-in surveillance tracking user activity.
  • It reinforces isolation by restricting all tools to state-approved ones.
  • It aligns with North Korea’s ideology of self-reliance (Juche).

Key Systems and Programs

IPTables

A firewall configured to block global internet access, prevent VPNs, restrict apps, and force traffic through government servers.

Why this matters: IPTables functions as a political firewall enabling censorship and monitoring.

SELinux

Locks down system files and protects government-monitoring tools.

Why this matters: SELinux enforces control and prevents system modification.

securityd

A daemon that reboots the system when protected files change.

Why this matters: This indicates constant system surveillance.

rtscan

A kernel module monitoring system resources for tampering.

Why this matters: Reduces privacy and blocks user modifications.

scnprc & opprc

Hidden monitoring daemons running by default.

libos.so.0.0.0

A core system file enforcing monitoring and reboot protections.

Startup & Init Config Files

Files such as /usr/share/autostart/scnprc.desktop automatically restart monitoring services on boot.

Installing Red Star OS in VMware Workstation Pro

Requirements

  • VMware Workstation Pro
  • Red Star OS ISO
  • 2–4 GB RAM
  • 20 GB disk
  • CPU virtualization enabled

1. Create a New VM

Open VMware → Create a New Virtual Machine.

2. Setup Type

Select Custom.

3. Select ISO

Load the ISO. If prompted, choose Linux → Other Linux 4.x kernel 64-bit.

4. Name the VM

  • Name: Red Star OS
  • Type: Linux Fedora 64-bit

5. VM Settings

  • Memory: 2–4 GB
  • CPU: 2 cores
  • Network: Host-Only (safer)
  • I/O Controller: LSI Logic
  • Disk Type: IDE → 20 GB → single file

Securing the System

Set Up Root User

su

Change Language to English (Optional)

sed -i 's/ko_KP/en_US/g' /etc/sysconfig/i18n /usr/share/config/kdeglobals
reboot

Remove IPTables

rm /etc/sysconfig/iptables

Disable SELinux

vi /boot/grub/grub.conf

Add to the kernel line:

selinux=0

Verify:

getenforce

Disable rtscan

python
>>> import fcntl
>>> fcntl.ioctl(open('/dev/res','wb'),29187)

Kill Monitoring Services

killall scnprc
killall opprc

Do NOT Reboot Until libos Is Replaced

Compile and Replace libos.so.0.0.0

gcc -c -Wall -Werror -fpic libos.c
gcc -shared -o libos.so.0.0.0 libos.o
cp ./libos.so.0.0.0 /usr/lib/libos.so.0.0.0
chown root:root /usr/lib/libos.so.0.0.0
chmod 755 /usr/lib/libos.so.0.0.0
ln -sf /usr/lib/libos.so.0.0.0 /usr/lib/libos.so.0

Reboot and Remove Startup Files

rm -f /usr/share/autostart/scnprc.desktop
rm -f /etc/init/ctguard.conf

Install Programs

cd /path/to/program
wine yourfile.exe

Conclusion

Red Star OS is difficult to modify due to limited documentation and protective systems designed to reboot or break when monitored components are altered. Securing it requires following precise steps and replacing files carefully.
